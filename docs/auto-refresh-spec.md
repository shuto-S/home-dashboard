# 自動更新仕様書

## 1. 目的

本ダッシュボードに表示する以下の情報を、利用者の明示操作なしで安定的に更新する。

- 現在日時
- 現在天気と時間帯別天気
- Google カレンダー予定

本仕様は、サーバレス構成かつ家庭内常時表示を前提とする。

## 2. 前提条件

- アプリはブラウザ上で動作する静的フロントエンドである
- 天気は Open-Meteo API を利用する
- カレンダーは Google Calendar API を利用する
- キャッシュ保存先は `localStorage` とする
- 表示端末はリビング等で常時表示されることを想定する

## 3. 対象データ

### 3.1 日時

- 現在日付
- 現在曜日
- 現在時刻

### 3.2 天気

- 指定地点の現在天気
- 当日の最高気温 / 最低気温
- 直近 12 時間の時間帯別天気
- 地点名

### 3.3 カレンダー

- Google カレンダー予定
- 表示対象期間は「現在時刻から 7 日先まで」とする
- 各予定には開始時刻と終了時刻を表示する
- 終日予定は `All Day` 表記とし、同日内の先頭に並べる

## 4. 更新仕様

### 4.1 日時更新

- 更新方式: クライアント内タイマー更新
- 更新間隔: 1 秒
- 備考: 秒は表示しなくても、分切り替えを確実に反映するため 1 秒更新を許容する

### 4.2 天気更新

- 更新方式: API ポーリング
- 初回表示時: 即時取得
- 定期更新間隔: 15 分
- オンライン復帰時: 即時再取得
- 手動更新: 任意で実行可能

### 4.3 カレンダー更新

- 更新方式: API ポーリング
- 初回表示時: 認証済みであれば即時取得
- 定期更新間隔: 10 分
- オンライン復帰時: 認証済みであれば即時再取得
- 手動更新: 任意で実行可能

## 5. キャッシュ仕様

### 5.1 保存対象

- 天気レスポンス整形後データ
- カレンダーレスポンス整形後データ
- Google OAuth アクセストークンと有効期限

### 5.2 起動時挙動

- 起動直後は `localStorage` 上のキャッシュを読み込む
- キャッシュが存在する場合は即座に画面表示する
- その後バックグラウンドで最新データを取得する

### 5.3 キャッシュ不整合時

- JSON 破損などで復元できない場合はキャッシュを無視する
- 画面は空状態またはローディング状態で継続する

## 6. 認証仕様

### 6.1 初回認証

- 利用者が Google 認証を実行する
- 認証成功後、アクセストークンと有効期限を保存する

### 6.2 再訪時

- 保存済みトークンが有効期限内であれば自動復元する
- 自動復元後は利用者操作なしでカレンダー取得を開始する

### 6.3 期限切れ時

- Google Calendar API が `401` を返した場合、既存トークンを破棄する
- サイレント再認証を 1 回試行する
- 再認証に成功した場合は自動的に再取得する
- 再認証に失敗した場合のみ再ログインを要求する

### 6.4 制約

- Google の認証ポリシーやセッション状態により、完全な無期限自動認証は保証できない
- サーバレス構成のため、リフレッシュトークンの安全な永続保持は行わない

## 7. エラー処理

### 7.1 天気取得失敗時

- キャッシュがあれば最後に取得した内容を表示し続ける
- 表示上は `Offline - last updated HH:MM` 等で状態を示す

### 7.2 カレンダー取得失敗時

- キャッシュがあれば最後に取得した内容を表示し続ける
- 認証エラー以外では即時ログアウト扱いにしない

### 7.3 ネットワーク断時

- 表示中データを保持したまま継続表示する
- ブラウザの `online` イベント受信時に再取得する

## 8. UI 要件

- 自動更新中であることを強く主張しない
- 更新状態は必要最小限のテキストで示す
- 手動更新ボタンは補助操作として控えめに表示する
- オフライン時もレイアウトが崩れず、最後の取得結果を読めること

## 9. 非機能要件

- 常時表示に耐える軽量動作であること
- API 呼び出し頻度は必要以上に高くしないこと
- ブラウザ再読み込み後も表示体験が大きく損なわれないこと
- モバイル幅でも UI が破綻しないこと

## 10. 将来拡張

- 深夜帯の更新頻度自動低下
- 画面非表示時のポーリング間隔延長
- 連続失敗時の指数バックオフ
- 複数カレンダー統合表示
- 地点検索 UI の追加

## 11. 受け入れ条件

- アプリ起動時、日時が自動表示される
- アプリ起動時、キャッシュがあれば即表示される
- 天気が 15 分ごとに自動更新される
- カレンダーが 10 分ごとに自動更新される
- ネットワーク復帰時に自動で再取得される
- 認証済みかつ有効なトークンがある場合、再ログイン不要でカレンダーが表示される
- 期限切れ時は可能な範囲で自動再認証を試みる
- API 失敗時でも最後の表示内容が維持される
